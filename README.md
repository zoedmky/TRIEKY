# TRIEKY

  <b>T</b>andem <b>R</b>epeat <b>I</b>nspector for <b>E</b>mergent <b>K</b>-mer <b>v</b>ariet<b>Y</b><br>
  <i>...because tandem repeats are tricky.</i>

</div>

TRIEKY is a lightweight, dependency-free post-processing tool for PacBio TRGT VCF files. It inspects reconstructed allele sequences to identify unexpected repeat motifs, detect interruption patterns, and summarize complex allele composition.

> ⚠️ **Disclaimer & Limitations**
> * **Research Use Only (RUO):** This tool is a prototype designed for research and exploratory purposes. It is not validated for primary clinical diagnosis.
> * **Proof of Concept:** Developed at the Biochemistry and Molecular Biology Department, Neurogenetics Laboratory, Hospices Civils de Lyon (France) to investigate atypical repeat motifs and interruption patterns in TRGT outputs.
> * **Validation Status:** TRIEKY has been evaluated on a limited number of samples. Results should be interpreted with caution. Feedback and contributions are welcome.

---

## Why TRIEKY?
TRGT is a powerful tandem repeat genotyping tool developed by PacBio. Its output is based on the motifs defined in the reference BED file used for genotyping. 
When an allele contains repeat motifs that are absent from the reference BED file, such as **interruption motifs** in *DMPK* or *HTT* or **non-canonical motifs** in *RFC1* (CANVAS), these motifs may remain hidden in the default VCF output. 

In clinical genetics, characterizing this "hidden" sequence composition can be important, as repeat interruptions or alternative motif conformations may influence pathogenicity, modify phenotype, or explain apparent discrepancies between genotype and clinical presentation.

TRIEKY was developed as an independent companion tool to provide additional insight into repeat composition directly from TRGT VCF outputs.

---

## Features

*   **Multi-locus analysis:** Process multiple TRGT loci simultaneously using a single configuration file.
*   **Flexible Configuration:** Tailor analysis per locus (thresholds, motif sizes, filters) via a TSV file.
*   **Automatic extraction of expected motifs**  Extract the motifs defined in the BED file directly from the TRGT VCF.
*   **Per-allele  analysis:** Analyzes and reports composition for each allele independently.
*   **Junction Filtering:** Automatically filters out artifactual k-mers generated by the boundaries of perfectly repeated expected motifs.
*   **Rotational Grouping:** Optionally groups equivalent motifs together (e.g., merging `CAG`, `AGC`, and `GCA` under the most frequent representative).
*   **Zero Dependencies:** Built entirely using the Python standard library.

---

## How it Works

TRIEKY extracts the reconstructed alternative sequences (`ALT`) from the TRGT VCF. Depending on the locus configuration, one of two complementary counting strategies is used.

### 1. Frame Mode
Designed for loci composed of a single repeat unit (e.g., *DMPK*, *HTT*, *FXN*). 
The sequence is analysed independently in every possible reading frame (0 to k-1), using non-overlapping k-mers.
This approach preserves the repeat structure while detecting unexpected motifs, making it well suited for simple tandem repeats.

### 2. Sliding Mode
Designed for more complex loci containing multiple repeat units or several motif lengths (e.g., *RFC1*). 
A sliding window scans every possible k-mer along the reconstructed allele sequence.
This exhaustive strategy allows the detection of complex or mosaic repeat structures that cannot be captured by a fixed reading frame.

## Usage 

To run TRIEKY, you need your TRGT VCF file and a configuration TSV file (`targets.tsv`) defining your loci of interest.

```bash
# Using standard Python
python trieky.py targets.tsv input.vcf output_report.txt

# Or using uv (recommended)
uv run python trieky.py targets.tsv input.vcf output_report.txt
```

---

## Configuration 

TRIEKY is configured using a simple TSV file (targets.tsv).

Each row defines the behaviour for one locus.

* **TRID** : Repeat identifier matching the TRGT VCF INFO field (e.g., DMPK).
* **motif_sizes** : Comma-separated list of motif lenghts to analyze, or leave empty to infer from VCF (e.g., 3 or 5,6).
* **startswith** : Restrict reported motifs to a specific first nucleotide (ANY, A, T, C, or G).
* **count_min** : Minimum number of occurrences required to report a motif (e.g., 5).
* **keep_only_unexpected** : true to hide motifs already expected by TRGT; false to report everything.
* **group_rotations** : true to merge rotationally equivalent motifs.
